# 阶段1：读侧数据源作用域解析

## 触发条件

任何 **read-side** Operation handler 即将调用 engine 查询 pages / chunks / links / search 之前；且该 handler 不自行实现 source 过滤（禁止 hand-roll）。

## 输入字段

| 字段 | 类型 | 来源 | 可选 |
|------|------|------|------|
| `ctx.auth.allowedSources` | `string[]` | OAuth federated read grant | 是 |
| `ctx.sourceId` | `string` | dispatch / CLI / token write scope | 否（必填，默认 `'default'`） |
| `ctx.remote` | `boolean` | transport | 否 |

Per-call 参数（仅部分 op，如 query）：`source_id`、`all_sources` — 由 `resolveRequestedScope` 单独处理，本阶段为 **无 per-call override** 时的默认 ladder。

## 判定规则

调用 `sourceScopeOpts(ctx)`，按**固定 precedence** 返回 engine opts fragment：

```
1. IF ctx.auth?.allowedSources EXISTS AND length > 0
     RETURN { sourceIds: allowedSources }

2. ELSE IF ctx.sourceId is set (always true after dispatch)
     RETURN { sourceId: ctx.sourceId }

3. ELSE
     RETURN {}
```

**空数组规则**：`allowedSources: []` **不**视为「读全库」；fall through 到步骤 2 使用 scalar `ctx.sourceId`。防止攻击者传 `[]`  widen scope。

**Link 读 op 附加规则**（get_links / get_backlinks）：在 `sourceScopeOpts` 结果上，若 `ctx.remote !== false` 且结果为 scalar `{ sourceId }` 无 `sourceIds`，则 **promote** 为 `{ sourceIds: [sourceId] }`，使 engine 走 federated 三分支 scope（from/to/origin 全过滤）。Trusted local 保留 scalar 跨 source 视图供 reconcile 内部工具使用。

## 状态读写位置

无持久化；纯函数从 `OperationContext` 派生 `{ sourceId? | sourceIds? }`，spread 进 engine 调用。

## 正常路径

1. Handler 收到 `ctx`
2. `const scope = sourceScopeOpts(ctx)` （link 读改用 `linkReadScopeOpts(ctx)`）
3. `engine.search({ ...params, ...scope })` 或等价 API
4. 返回结果

## 分支路径

| 条件 | 行为 |
|------|------|
| federated 非空数组 | engine 用 `sourceIds[]` IN 过滤 |
| 仅 scalar sourceId | engine 用单 source 过滤（link local 例外见上） |
| `{}` 且无 per-call resolver | local CLI 历史行为：engine 不 filter（全 brain）；remote 场景不应落在此分支——remote 应用 `resolveRequestedScope` 禁止 widen |
| per-call `__all__` + remote | resolver 忽略 `__all__`，返回 grant（`sourceScopeOpts`），非 `{}` |
| per-call explicit source_id 不在 grant | `OperationError permission_denied` |

## 失败处理

- engine 查询错误：向上抛，dispatch 包装 JSON error
- 无「scope 解析失败」分支——precedence 总产出确定结构

## 幂等性 / 一致性约束

- 同 ctx 多次调用产出相同 scope fragment
- Handler **禁止**在 spread scope 后再忽略或覆盖为 unscoped 查询
- 后续阶段只能使用已 scope 的结果集，不能为「补全」再开 unscoped 二次查询（除非 trusted local 且文档化）

## 代码骨架

```typescript
export function sourceScopeOpts(ctx: OperationContext): {
  sourceId?: string;
  sourceIds?: string[];
} {
  const allowed = ctx.auth?.allowedSources;
  if (allowed && allowed.length > 0) return { sourceIds: allowed };
  if (ctx.sourceId) return { sourceId: ctx.sourceId };
  return {};
}

export function linkReadScopeOpts(ctx: OperationContext) {
  const scope = sourceScopeOpts(ctx);
  if (ctx.remote !== false && scope.sourceId && !scope.sourceIds) {
    return { sourceIds: [scope.sourceId] };
  }
  return scope;
}
```

## 最小验证清单

- federated `['a','b']` → engine 收到 `sourceIds: ['a','b']`，不含 c source 行
- `allowedSources: []` + `sourceId: 'x'` → `{ sourceId: 'x' }`，非全库
- remote + link read + scalar only → `{ sourceIds: ['x'] }`
- local + link read + scalar → `{ sourceId: 'x' }` 保持 scalar
- 两 handler 同 ctx 产出 identical scope（无 hidden mutation）
